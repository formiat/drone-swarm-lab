# DRONE_B.9.linear — Линейный план после DRONE_B.8

Дата фиксации: 2026-05-23

## Исходная точка

Все пять stages из DRONE_B.8 реализованы. Детали развилки — в DRONE_B.9.branches.md.
Этот документ выстраивает ветки в линейную последовательность по зависимостям,
без необходимости выбирать направление заранее.

## Линейный план

```
Шаг 1: Интеграция Safety Layer в аллокаторы
Шаг 2: End-to-end SITL прогон
Шаг 3: Visualization / Replay UI
Шаг 4: Новый алгоритмический слой
```

### Шаг 1 — Интеграция Safety Layer в аллокаторы

Safety Layer реализован в `swarm-safety`, но `filter_safe_tasks` нигде не вызывается
внутри аллокаторов. Задачи в no-fly зонах по-прежнему назначаются.

Что сделать:

- Добавить `safety_config: Option<SafetyConfig>` в `AllocationContext`
  (или как параметр в каждый аллокатор).
- Вызвать `filter_safe_tasks` перед выбором задачи в Greedy, CBBA, Auction,
  Centralized и ConnectivityAware аллокаторах.
- Добавить integration-тест: coverage сценарий с no-fly зоной в центре сетки —
  ни одна задача внутри зоны не назначается.
- Добавить `safety_violations` счётчик в `AggregateMetrics` и экспорт.

Done: аллокаторы не назначают задачи в запрещённых зонах.

### Шаг 2 — End-to-end SITL прогон

MAVLink транспорт и `sitl_agent` binary реализованы, MockMavlink работает.
Реального прогона с PX4 не было — M17 формально не закрыт.

Что сделать:

- Поднять PX4 SITL (Gazebo).
- Запустить `sitl_agent` с coverage сценарием через `--scenario-suite`.
- Убедиться что цепочка Task → Waypoint → MAVLink → PX4 работает на одном агенте.
- Добавить `docs/SITL_SETUP.md` с воспроизводимой инструкцией.
- После одного агента — попытка запустить двух агентов одновременно.

Done: агент пролетает по waypoints из coverage.ideal.json через реальный PX4 SITL.

### Шаг 3 — Visualization / Replay UI

Сейчас весь вывод — CSV/JSON/Markdown таблицы. Поведение агентов не видно глазами.

Что сделать:

- CLI-утилита `replay` (`swarm-examples/src/bin/replay.rs`):
  читает JSONL event log, рисует ASCII-grid с позициями агентов по тикам.
- Команды: `--log run.jsonl`, `--tick N`, `--follow` (live режим).
- Поддержка BeliefMap overlay (показывать posterior по ячейкам в SAR сценариях).
- Поддержка InspectionGraph overlay (показывать покрытые/непокрытые рёбра).

Полноценный вариант (egui / Bevy) — как отдельный следующий шаг после ASCII версии.

Done: `cargo run --bin replay -- --log run.jsonl` показывает анимацию сценария.

### Шаг 4 — Новый алгоритмический слой

Открытый исследовательский слой поверх стабильной интегрированной системы.
Конкретное направление выбирается по интересу в момент начала шага.

Кандидаты:

- Динамическое перераспределение при отказах: агент упал — его bundle
  переаллоцируется без полного перезапуска CBBA консенсуса.
- Multi-robot path planning с учётом Safety Layer: геометрические ограничения
  влияют на маршруты, не только на аллокацию.
- Оптимизация маршрутов для Infrastructure Inspection: 2-opt или OR-Tools
  вместо жадного nearest-neighbour TSP.
- Многоуровневая координация: группы агентов с локальным лидером и глобальным
  координатором.

Done: определяется при выборе конкретного направления.
