# DRONE_B.12 — Варианты дальнейшего развития

Дата фиксации: 2026-05-23

## Что есть сейчас

- 5 стратегий аллокации
- 4 миссии (Coverage, EmergencyMesh, SAR v2, Inspection)
- Safety Layer (standalone, не интегрирован в аллокаторы)
- CBBA Robustness (TSP, retransmission, convergence metrics)
- SITL scaffold (mock работает, реальный PX4 — нет)
- Replay CLI (ASCII, текстовый summary)

## Варианты

### Ветка A — Починка алгоритмических дыр

Это не полировка — содержательные баги:

- `success=0.0` при `edge_coverage=1.0` в inspection — метрика сломана.
- CBBA и centralized дают 0% на SAR — grid_cell handling не работает с этими стратегиями.

Без этого benchmark сравнение стратегий методологически нечистое.

### Ветка B — Новая миссия

Проект умеет строить новые миссии через DSL. Интересные варианты:

- **Wildfire / flood mapping** — динамическая карта угрозы, агенты перераспределяются
  по мере развития ситуации.
- **Multi-target pursuit** — цели движутся, агенты преследуют.
- **Logistics / delivery** — задачи с pickup + dropoff, зависимости между задачами.

Каждая потребует нового класса задач или новой механики.

### Ветка C — Алгоритмическая глубина

- Динамическое перераспределение при отказе агента без полного перезапуска CBBA.
- Многоуровневая координация: группы с локальным лидером + глобальный координатор.
- Communication-optimal allocation: минимизировать трафик при заданном качестве аллокации.

### Ветка D — Реальный SITL

Подключить настоящий `MavlinkTransport` в `sitl_agent`. Сейчас `--connection` парсится,
но не используется. Один агент летит по waypoints через PX4 SITL.

### Ветка E — Визуализация

egui или Bevy — интерактивный просмотр replay с BeliefMap, InspectionGraph, позициями
агентов. Сейчас есть только ASCII.

## Зависимости и совместимость

```
Ветка A (баги)        — независима, желательна перед публикацией любого результата
Ветка B (новая миссия) — независима от остальных
Ветка C (алгоритмы)   — лучше после A (чтобы benchmark был корректным)
Ветка D (SITL)        — независима, нужна только установка PX4
Ветка E (визуализация) — независима, полезнее после стабильных replay schemas
```

## Линейный вариант (если не хочется выбирать)

С акцентом на алгоритмическую чистоту:

```
A (баги) → C (алгоритмы) → B (новая миссия) → E (визуализация) → D (SITL)
```

С акцентом на исследовательский интерес:

```
A (баги) → B (новая миссия) → C (алгоритмы) → E (визуализация)
```
